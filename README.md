# User Stories
As a student, I want to be able to manage my time and tasks in one place.
As a member of my campus community I want to be able to communicate with my peers and be aware of campus events.

# Project Description

CampusConnect is a web application built with Django that helps students keep track of their weekly tasks while also seeing events happening around campus. The goal of the app is to give students a simple place where they can manage their schedule and stay updated on campus activities.


The application follows the CRUD model, meaning users can create, read, update, and delete their own tasks and events. This project is being developed for our Software I course and is meant to simulate a real development process where we plan features, work in sprints, and collaborate using GitHub.


# Problem Statement


Students usually use multiple apps to manage assignments, reminders, and social events. This can make it harder to stay organized. CampusConnect brings these things together in one place so students can track their responsibilities while also discovering events happening on campus.


# Project Objectives


The main goal of this project is to practice building a full web application from start to finish. We are using Django to build the system and GitHub to manage our code and collaboration. Throughout the project we are following an Agile style workflow where we break the work into weekly sprints and present our progress.


# Technology Stack


Backend: Django with Python
Frontend: HTML and CSS using Django templates
Database: SQLite for development
Version control: Git and GitHub
Environment: Python virtual environment


# Core Features


User Accounts
Users can create an account, log in, update their profile information, and delete their account if needed.


Task Manager
Users can add weekly tasks, view their task list, edit tasks, and delete them once they are completed or no longer needed.


Event Calendar
Users can create events, view events posted by others, edit events they created, and delete them.


# System Structure


The project follows Django’s Model View Template structure.


Models handle the database structure
Views handle the logic of the application
Templates display the information to the user


To keep things organized, the project is separated into different Django apps such as users, tasks, and events.
